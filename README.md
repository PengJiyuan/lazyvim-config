# LazyVim 完整上手教程

这份教程是给你当前这套配置写的，目标不是把 Vim 全讲完，而是让你能用 **LazyVim** 真实完成前端开发工作。

内容分三部分：

1. 从零入门
2. VS Code 对照表
3. 前端开发实战工作流

---

# 1. 先理解一个最重要的概念

在 VS Code 里，你打开编辑器就能直接输入。

在 Neovim / LazyVim 里，不是这样。

它最常用的是三种模式：

## 1.1 Normal mode（普通模式）
默认模式。

作用：
- 移动光标
- 删除、复制、粘贴
- 跳转定义
- 打开文件树
- 查找文件
- 执行各种命令

这个模式下，很多按键不是“输入文字”，而是在“执行动作”。

---

## 1.2 Insert mode（输入模式）
这个模式下你才能像 VS Code 那样正常打字。

进入方式：
- `i`：在当前光标前开始输入
- `a`：在当前光标后开始输入
- `o`：在下一行新开一行并开始输入

你先只记：
- `i` = 开始输入

---

## 1.3 Visual mode（选择模式）
用来选择文本。

进入方式：
- `v`

退出：
- `Esc`

---

## 1.4 最重要的习惯
如果你发现：
- 按键没法输入文字
- 光标乱跳
- 按一个键删了一行

先不要慌，大概率只是因为你现在在 **Normal mode**。

解决办法：
- 想打字就按 `i`
- 乱了就按 `Esc`

# 6. 你配置里的特殊前缀：Leader

你现在最容易看不懂的是 `<leader>`。

## 6.1 Leader 是什么
Leader 不是一个字母，而是一个“前缀键”。

你当前这套 LazyVim 默认：
- `<leader>` = `Space`

所以：
- `<leader>fe`

真正的按法不是同时按，而是顺序按：
- `Space`
- `f`
- `e`

---

## 6.2 其他缩写是什么意思
- `<leader>` = `Space`
- `<C-p>` = `Ctrl + p`
- `<D-p>` = `Command + p`

说明：
- `C` = Ctrl
- `D` = macOS 上的 Command（⌘）

例如：
- `<leader>xp` = `Space x p`
- `<C-p>` = `Ctrl + p`
- `<D-p>` = `⌘ + p`

注意：
在 iTerm2 里，`⌘` 键不一定总能稳定传给 Neovim，所以 `Ctrl + p` 往往更可靠。

# 9. 前端开发实战工作流

下面这部分只讲你最常见的前端开发操作。

---

## 9.1 打开一个项目后，怎么找文件

### 方法 A：快速找文件
按：
- `Ctrl + p`

输入文件名，例如：
- `App.tsx`
- `index.tsx`
- `useUser.ts`
- `tailwind.config`

按回车打开。

---

### 方法 B：用文件树看项目结构
按：
- `Space f e`

你现在这套文件树会：
- 在左边显示
- 比较稳定
- 显示 hidden 文件
- 显示 ignored 文件

适合用来查看：
- `src/`
- `components/`
- `pages/`
- `.env`
- `package.json`
- `tailwind.config.*`

---

## 9.2 怎么开始改代码

1. 找到文件
2. 按 `i`
3. 开始输入
4. 输入完按 `Esc`
5. 保存用 `:w`

---

## 9.3 在多个文件之间切换

### 当前打开文件列表
按：
- `Space Space`

### 最近文件
按：
- `Space f r`

### 推荐你最常用的切文件组合
- `Ctrl + p` = 找任意文件
- `Space Space` = 当前打开的文件之间切
- `Space f r` = 最近文件

---

## 9.4 跳到定义
把光标放在函数、变量、组件、hook 上，按：
- `g d`

### 跳回来
按：
- `Ctrl + o`

这个很重要。

---

## 9.5 看引用
把光标放在符号上，按：
- `g r`

适合查：
- 一个组件被哪里用到
- 一个函数被哪里调用
- 一个类型在哪里被引用

---

## 9.6 重命名
把光标放在符号上，按：
- `Space r n`

输入新名字，回车。

适合：
- 变量重命名
- 函数重命名
- hook 重命名
- 组件重命名

---

## 9.7 看类型 / 文档 / 注释
把光标放在符号上，按：
- `Shift + k`

也就是：
- `K`

这个对 TypeScript 很有用。

---

## 9.8 修复错误 / Quick Fix
把光标放在问题上，按：
- `Space c a`

适合：
- 自动导入
- quick fix
- 某些 LSP action

---

## 9.9 看报错
按：
- `Space x p`

作用：
- 打开 Problems
- 看 TypeScript / ESLint / LSP 报错

建议你改完一段代码之后就看一次。

---

## 9.10 开终端
按：
- `Space t t`

可以在里面跑：
- `npm run dev`
- `pnpm dev`
- `npm test`
- `pnpm lint`

### 从终端返回普通模式
按：
- `Esc`
- `Esc`

---

# 10. 真实的一次前端开发流程示例

假设你要修改一个页面里的按钮和跳转逻辑。

## 第 1 步：找到文件
按：
- `Ctrl + p`

输入：
- `SettingsPage.tsx`

回车。

- `/Button`

回车后：
- `n` = 下一个匹配

---

按：
- `i`

改完后：
- `Esc`

保存：
- `:w`

---

## 第 4 步：查看函数定义
把光标放在 `onClick` 对应函数上，按：
- `g d`

看完返回：
- `Ctrl + o`

---

## 第 5 步：重命名函数
把光标放在函数名上，按：
- `Space r n`

输入新名字，回车。

---

## 第 6 步：看有没有错误
按：
- `Space x p`

---

把光标放在报错位置，按：
- `Space c a`

看看有没有自动修复。

---

按：
- `Space Space`

在打开的 buffers 里切换。

# 11. 你最容易卡住的地方

## 11.1 按键后不能输入
原因：
你还在普通模式。

解决：
- 按 `i`

---

## 11.2 按键后东西乱跳
原因：
普通模式下很多键是命令，不是打字。

解决：
- 乱了先按 `Esc`

---

## 11.3 不知道 leader 是什么
记住：
- `<leader>` = `Space`

---

## 11.4 不知道组合键是不是同时按
像这样的：
- `<leader>fe`

不是同时按。

而是顺序按：
- `Space`
- `f`
- `e`

---

## 11.5 `⌘ + p` 不稳定
原因：
iTerm2 对 `Command` 键支持不一定稳定。

建议优先用：
- `Ctrl + p`
- `Space Space`
- `Space f e`
